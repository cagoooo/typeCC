# 中文注音打字遊戲 (TypeCC) - 開發進度與未來優化建議報告

本文件旨在追蹤「中文注音打字遊戲 (typeCC)」的開發進度，並提供未來深度優化與進階功能開發的詳細藍圖參考。

---

## ✅ 目前已完成進度 (至 v1.2.1)

在近期的開發中，專案已成功從單一 HTML 檔案升級為現代化前端架構，並實裝了許多核心功能：

### 1. 系統架構與部署
*   **Vite 現代化建構**：成功模組化程式碼 (`main.js`, `audio.js`, `style.css`)，大幅提升開發與維護效率。
*   **GitHub Actions CI/CD**：實作自動化部署流程，推送到 `main` 分支後自動編譯並發布至 GitHub Pages。
*   **PWA 漸進式網頁應用**：支援 Service Worker 離線快取與桌面/手機端安裝，並修復了最新的圖示解析度問題。

### 2. 遊戲核心體驗
*   **多階段難度系統**：
    *   **初學者模式 (Beginner)**：為國小低年級設計，極慢速、顯示按鍵提示與完整注音。
    *   **入門模式 (Easy)**：速度適中，適合剛學鍵盤的使用者。
    *   **進階模式 (Medium)**：隱藏按鍵提示，考驗直覺反應。
    *   **大師模式 (Hard)**：高速掉落，具備極大挑戰性。
*   **動態生成控制**：以定時生成器取代原始的固定補位機制，使畫面字元密度更合理且不壓迫。
*   **音效反饋系統**：加入打擊音效、遊戲結束音以及最後 10 秒倒數警告，強化遊戲沉浸感。

### 3. 社群與擴充功能
*   **Firebase 整合**：支援匿名登入，並實裝即時更新的「全球排行榜 (Firestore)」，穩定性已大幅優化。
*   **成績截圖分享**：整合 `html2canvas`，結束後可一鍵生成精美滿意度成績圖片供社群分享。
*   **進階大數據詞庫**：擴充至數百字，並新增常用四字成語，增加打字的教育價值與變化性。

---

## 🚀 未來優化改良與擴展開發建議

專案基礎已相當穩固，未來可朝向「遊戲化 (Gamification)」、「教育精緻化 (Educational Value)」、「UI/UX 聲光效果」三大方向深度發展：

### 📈 第一階段：教育與個人化能力深化
1.  **個人錯題本與重點複習**：
    *   **功能描述**：後台記錄玩家最常打錯或漏打的字詞。
    *   **應用**：遊戲結束時提供「錯題練習模式」，讓使用者專門針對不熟悉的注音組合進行特訓。
2.  **自訂詞庫 (教師/家長模式)**：
    *   **功能描述**：允許使用者上傳 CSV/JSON 檔，或在介面輸入自訂的單字與成語陣列。
    *   **應用**：國小老師可以把「本週生字表」匯入遊戲中，讓學生邊玩邊準備聽寫測驗。
3.  **學習曲線學習成效報告**：
    *   **功能描述**：除了全球排行，建立個人長期 WPM (Words Per Minute) 與正確率的歷史折線圖。

### 🎮 第二階段：遊戲化元素與社群互動
1.  **道具與連擊系統 (Combo System)**：
    *   **連續擊殺加成**：連續不打錯字時累積 Combo，分數呈倍數成長，中斷則歸零。
    *   **掉落特殊道具**：例如「全畫面凍結 (時間暫停 3 秒)」、「炸彈 (清除畫面上所有字元取得基礎分)」，增加趣味性。
2.  **段位與成就系統**：
    *   **稱號解鎖**：達成特定目標（例如：連續 50 個字不錯、總分突破一萬）獲得「鍵盤學徒」、「打字大師」等稱號。
3.  **非同步對戰 / 多人房間功能 (進階)**：
    *   **功能描述**：利用 Firebase Realtime Database 實現建立房間碼。
    *   **應用**：兩位玩家同時面對相同的掉落字序列，比拼誰撐得久或分數高，畫面兩側顯示對手的進度條。

### ✨ 第三階段：UI/UX 與視覺聲光表現
1.  **主題外觀切換 (Themes)**：
    *   加入深色模式 (Dark Mode) 與淺色模式 (Light Mode) 自由切換。
    *   設計「兒童專屬」主題（例如：宇宙星空、海洋世界背景，字元變成外星人或泡泡）。
2.  **微動畫與粒子特效**：
    *   當字元被擊中時，除了現在的縮放消失，可加入更華麗的「碎裂粒子」動畫 (可使用輕量級 canvas 庫)。
    *   當達成 10 連擊以上時，輸入框與計分板發出動態輝光。
3.  **語音朗讀輔助 (TTS)**：
    *   結合 Web Speech API，當顯示該字時，語音會念出該字的發音，同時鍛鍊「聽打」與「看打」能力。

### ⚙️ 第四階段：技術與架構升級 (維持高品質)
1.  **測試自動化層級提升**：
    *   引入 Playwright 或 Cypress 進行 E2E (端對端) 瀏覽器自動化測試，確保鍵盤事件、Firebase 資料寫入不發生回歸錯誤 (Regression)。
2.  **Google Analytics 4 深度事件追蹤**：
    *   埋設自訂事件追蹤，分析哪個「難度」最多人玩、平均「存活時間」，以及使用者最容易中途流失的環節，作為難度平衡參考依據。
3.  **SSR / 靜態預先渲染 (預留方案)**：
    *   若未來專案擴展為多頁面 (例如：首頁、獨立排行榜頁、關於我們)，可考慮遷至 Nuxt.js 或 Next.js 以便最佳化 SEO。
