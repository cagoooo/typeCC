# 中文注音打字遊戲 (TypeCC) - 開發進度與未來優化建議報告

本文件旨在追蹤「中文注音打字遊戲 (typeCC)」的開發進度，並提供未來深度優化與進階功能開發的詳細藍圖參考。

---

## ✅ 目前已完成進度 (至 v1.4.0)

在近期的開發中，專案已成功從單一 HTML 檔案升級為現代化前端架構，並實裝了許多核心功能與視覺聲光大幅升級：

### 1. 系統架構與部署
*   **Vite 現代化建構**：成功模組化程式碼 (`main.js`, `audio.js`, `style.css`)，大幅提升開發與維護效率。
*   **GitHub Actions CI/CD**：實作自動化部署流程，推送到 `main` 分支後自動編譯並發布至 GitHub Pages。
*   **PWA 漸進式網頁應用**：支援 Service Worker 離線快取與桌面/手機端安裝，並修復了最新的圖示解析度問題。
*   **安全性強化**：修正 Firebase API Key 遺失 localhost 存取權限的 403 錯誤，並確保跨域請求安全。

### 2. UI/UX 與聲光視覺大升級 (v1.4 最新亮點)
*   **主題外觀系統 (Themes)**：支援 `data-theme` 切換機制 (包含深色 Dark、淺色 Light、專為兒童設計的海洋泡泡風格 Kids-Ocean)。
*   **微動畫與連擊特效**：
    *   實作輕量級「碎裂粒子」動畫，擊中字元時有爆破視覺回饋。
    *   新增 Combo 系統，連續擊中累積 Combo 計數，達 10 連擊以上輸入框與計分板會發出動態輝光 (`.combo-glow`)。
*   **語音朗讀輔助 (TTS)**：結合 Web Speech API，成功打中中文字後系統會自動準確地唸出該字發音，強化聽打與認知能力。
*   **沉浸式背景音樂 (BGM)**：實作隨機 BGM 播放器，引入多首輕快充滿童趣的免版權音樂，並已將音源本地化 (`bgm1.mp3`, `bgm2.mp3`) 解決外部 CDN 防盜鏈阻擋問題。
*   **遊戲流程巧思**：遊戲開始時自動 focus 輸入框、防呆的注音與英文雜訊過濾比對，大幅提升順暢度。

### 3. 遊戲核心與多階段難度系統
*   **初學者模式 (Beginner)**：為國小低年級設計，極慢速、顯示按鍵提示與完整注音 (已進一步調降掉落頻率與數量，更貼近新手)。
*   **入門模式 (Easy)**：速度適中，適合剛學鍵盤的使用者。
*   **進階模式 (Medium)**：隱藏按鍵提示，考驗直覺反應。
*   **大師模式 (Hard)**：高速掉落，具備極大挑戰性。

### 4. 社群與教育大數據
*   **Firebase 整合**：支援匿名登入，並實裝即時更新的「全球排行榜 (Firestore)」。
*   **成績截圖分享**：整合 `html2canvas`，結束後可一鍵生成精美滿意度成績圖片供社群分享。
*   **進階詞庫**：內建數百字完整詞庫並支援常用四字成語，深具教育價值。

## 🚀 未來深度優化改良與擴展開發建議 (Roadmap)

隨著基礎體驗 (聲光、UI、流暢度) 已達高水準，未來建議朝向「教育個人化」、「社群電競化」、「跨裝置適配」三大主軸深入發展：

### 📈 第一階段：教育精準化與個人學習報告
1.  **個人錯題本與針對性複習**：
    *   **概念**：後台自動記錄玩家最常打錯、或落地漏接的字詞。
    *   **應用**：在開始選單新增「錯題特訓模式」，系統專門生成玩家較弱的注音組合（例如常把 ㄣ 和 ㄥ 搞混），進行弱點強化。
2.  **教師/家長自訂詞庫系統**：
    *   **概念**：設計一個專屬的管理員介面，或允許上傳純文字/CSV 檔。
    *   **應用**：讓國小老師可以直接把「本課生字表」或「本週成語」匯入成專屬關卡，並產生「專屬房間連結」派發給學生當作回家作業。
3.  **學習成長曲線 (Dashboard)**：
    *   **概念**：除了現有的單次全球排行，新增個人獨立的 Dashboard。
    *   **應用**：繪製長期的 WPM (每分鐘字數)、正確率、Combo 最高紀錄的歷史折線圖，讓孩子能看到自己的進度而獲得成就感。

### 🎮 第二階段：遊戲化元素 (Gamification) 與深度黏著
1.  **進階掉落物與干擾/增益機制 (Items)**：
    *   **特殊道具字**：畫面上偶爾飄下帶有光環的字，打字消除後可獲得「時間暫停 (凍結畫面 3 秒)」、「全頻核彈 (瞬間清空所有字並得分)」、「護盾 (抵銷一次落地扣分)」等技能。
    *   **魔王字 (Boss Word)**：需要連續打對 3 次才能消除的巨大字元。
2.  **成就解鎖與段位系統**：
    *   **概念**：將遊戲體驗目標化。設定如「無失誤通關大師模式」、「累積消除 10,000 字」、「最高 50 連擊」等成就。
    *   **應用**：解鎖後可獲得特殊的「玩家稱號」顯示在結算畫面，或是解鎖隱藏版的外觀主題/背景音樂。
3.  **即時對戰 / 同步大廳 (PvP Mode)**：
    *   **概念**：利用 Firebase Realtime Database 或 WebRTC 實作對戰室。
    *   **應用**：加入房間後，雙方玩家面對完全相同的字元掉落序列，畫面上方顯示拔河條 (Tug of War)，打字速度快的會將壓力推給對手 (對手畫面掉落速度變快)，增加刺激的電競感。

### ✨ 第三階段：跨世代操作體驗與互動 UI
1.  **虛擬鍵盤支援 (平板/手機適配)**：
    *   **概念**：目前打字遊戲高度依賴實體鍵盤，使得 iPad 等無實體鍵盤的平板族群難以遊玩。
    *   **應用**：於畫面下方開發專屬的「螢幕虛擬注音鍵盤」，並支援多點觸控，讓遊戲範圍打破桌機限制，邁向行動教育市場。
2.  **Live2D / 互動吉祥物伴學**：
    *   **概念**：在遊戲視窗角落放置一個動態吉祥物 (例如貓咪或機器人)。
    *   **應用**：隨著玩家打擊 Combo 增加，吉祥物會做出歡呼、跳躍的動畫；若漏字則會露出驚訝或鼓勵的表情，大幅提升幼童陪伴感。
3.  **全螢幕沉浸模式 (Fullscreen API)**：
    *   加上全螢幕展開按鈕，消除瀏覽器 UI 選單干擾，讓學生能在學校電腦教室享有更專注的遊玩體驗。

### ⚙️ 第四階段：技術底層架構升級
1.  **會員認證系統整合 (Firebase Auth)**：
    *   從現在的「匿名登入 (Anonymous)」進階到支援 Google / Email 綁定登入，確保玩家換了裝置依然能保留排名與稱號。
2.  **自動化測試 E2E (End-to-End Test)**：
    *   導入 Playwright 或 Cypress，撰寫自動化腳本模擬玩家打字與送出積分，確保未來每次功能大改版都不會破壞核心邏輯。
3.  **Google Analytics 4 事件追蹤 (GA4)**：
    *   精細化追蹤玩家的「死亡關卡」、「最常選擇的難度」、「平均遊玩時長」，以此作為未來難度數值調整 (Game Balance) 的大數據依據。
